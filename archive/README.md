# Archive

Historical files preserved for reference. Active development uses files in parent directory.

## Structure

- **catalogs/** - Previous versions of attack_catalog.json (v1-v4, combined)
- **testers/** - Legacy tester implementations before unified_tester_v3
- **planning-docs/** - Development planning, research notes, and design documents
- **test-runs/** - Log files and sample outputs from test runs

## Current Active Files

- `attack_catalog_v5.json` - Current attack catalog
- `tools/unified_tester_v3.py` - Current unified tester
- `tools/llm_judge_v2.py` - Current LLM judge

## Why Archive?

These files document the evolution of the project. While git history preserves everything, having them accessible makes it easier to:
- Reference old approaches
- Compare implementations
- Understand design decisions
