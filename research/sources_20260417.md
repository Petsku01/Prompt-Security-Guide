# Prompt Security Research Sources
**Collected:** 2026-04-17  
**Focus:** LLM security, prompt injection defense, AI safety — defense & research only

## 1. ClawGuard: A Runtime Security Framework for Tool-Augmented LLM Agents Against Indirect Prompt Injection
**URL:** https://arxiv.org/abs/2604.11790  
**Summary:** Introduces ClawGuard, a runtime security framework that enforces user-confirmed rule sets at every tool-call boundary in agentic LLM systems, converting alignment-dependent defense into a deterministic, auditable mechanism. It addresses three attack channels: web/local content injection, MCP server injection, and skill file injection. Evaluated across five SOTA models on AgentDojo, SkillInject, and MCPSafeBench.  
**Why relevant:** Directly relevant — a concrete defense framework for indirect prompt injection in tool-augmented agents with empirical evaluation. Covers MCP-based injection, which is current and practical.

## 2. Prompt Injection Attacks on Agentic Coding Assistants: A Systematic Analysis (SoK)
**URL:** https://arxiv.org/abs/2601.17548  
**Summary:** Systematization of Knowledge paper analyzing prompt injection attacks on agentic coding assistants (Claude Code, Copilot, Cursor, MCP-based systems). Proposes a three-dimensional taxonomy (delivery vectors, attack modalities, propagation behaviors). Meta-analysis of 78 studies (2021–2026) finds >85% attack success rates against SOTA defenses with adaptive strategies. Catalogs 42 attack techniques and analyzes 18 defense mechanisms, finding most achieve <50% mitigation. Concludes prompt injection must be treated as a first-class vulnerability class requiring architectural-level mitigations.  
**Why relevant:** Most comprehensive taxonomy and survey available. Directly maps the defense landscape and identifies where current mitigations fail — essential for understanding what defenses work and why.

## 3. Why LLM Safety Guardrails Collapse After Fine-tuning: A Similarity Analysis
**URL:** https://arxiv.org/abs/2506.05346  
**Summary:** Investigates why safety guardrails degrade after downstream fine-tuning, focusing on representation similarity between upstream alignment datasets and downstream fine-tuning tasks. High similarity between these datasets significantly weakens guardrails; low similarity yields more robust models with harmfulness score reductions up to 10.33%. Offers actionable dataset-design insights for fine-tuning providers.  
**Why relevant:** Critical for understanding when and why alignment defenses fail — directly informs how to construct more durable safety guardrails at the data-design level.

## 4. TraceSafe: A Systematic Assessment of LLM Guardrails on Multi-Step Tool-Calling Trajectories
**URL:** https://arxiv.org/abs/2604.07223  
**Summary:** Introduces TraceSafe-Bench, the first benchmark for mid-trajectory safety assessment in agentic LLM workflows. Covers 12 risk categories and 1,000+ execution instances. Evaluates 13 LLM guard models and 7 specialized guardrails. Key findings: (1) guardrail efficacy is driven by structural data competence (JSON parsing) more than semantic safety alignment; (2) architecture matters more than scale; (3) accuracy is resilient across extended trajectories.  
**Why relevant:** First systematic evaluation of guardrails in multi-step agent execution — reveals that structural reasoning is as important as safety alignment for effective defense. Directly applicable to designing guardrails for agentic systems.

## 5. PromptGuard: A Structured Framework for Injection-Resilient Language Models
**URL:** https://www.nature.com/articles/s41598-025-31086-y  
**Summary:** Presents a modular four-layer defense framework: input gatekeeping, structured prompt formatting, semantic output validation, and adaptive response refinement (ARR). Combines regex and MiniBERT-based detection to identify and block adversarial inputs. Published in Nature Scientific Reports.  
**Why relevant:** Peer-reviewed, practical defense framework integrating multiple defense layers. The structured pipeline approach is directly applicable to implementation guidance.

## 6. Image-Based Prompt Injection: Hijacking Multimodal LLMs Through Visually Embedded Adversarial Instructions
**URL:** https://labs.cloudsecurityalliance.org/research/csa-research-note-image-prompt-injection-multimodal-llm-2026/  
**Summary:** Cloud Security Alliance research note (March 2026) analyzing attack classes and threat scenarios for image-based prompt injection in multimodal LLMs. Identifies primary attack vectors and available countermeasures for visually-embedded adversarial instructions.  
**Why relevant:** Addresses the emerging multimodal attack surface — critical as LLMs increasingly process images. Industry-backed research with practical threat analysis.

## 7. LLM01:2025 Prompt Injection — OWASP Gen AI Security Project
**URL:** https://genai.owasp.org/llmrisk/llm01-prompt-injection/  
**Summary:** OWASP's official risk classification for prompt injection in generative AI. Defines prompt injection as the #1 LLM risk, covering both direct and indirect injection. Industry-standard reference for threat classification and recommended mitigations.  
**Why relevant:** Authoritative industry standard — essential baseline reference for any prompt security work. Defines the canonical threat model used across the industry.

## 8. A One-Prompt Attack That Breaks LLM Safety Alignment
**URL:** https://www.microsoft.com/en-us/security/blog/2026/02/09/prompt-attack-breaks-llm-safety/  
**Summary:** Microsoft Security Blog research showing that even minimal downstream fine-tuning can weaken LLM safety safeguards. Demonstrates a single-prompt attack that bypasses alignment, raising concerns about how reliably safety holds as models are adapted.  
**Why relevant:** Microsoft-backed research confirming alignment fragility — practical demonstration of why defense-in-depth is necessary and why single-layer defenses are insufficient.

## 9. Prompt Injections: The Inherent Threat to Generative AI (CIS White Paper)
**URL:** https://www.cisecurity.org/insights/white-papers/prompt-injections-the-inherent-threat-to-generative-ai  
**Summary:** Center for Internet Security white paper framing prompt injection as an inherent, escalating threat to government organizations. Provides guidance on defending SLTT (State, Local, Tribal, Territorial) government AI deployments against prompt injection.  
**Why relevant:** Policy/governance perspective — important for understanding regulatory and organizational defense approaches beyond purely technical solutions.
