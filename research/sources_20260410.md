## ArXiv Papers (Verified)

### 1. SmoothLLM: Defending Large Language Models Against Jailbreaking Attacks
- **URL:** https://arxiv.org/abs/2310.03184
- **Summary:** Proposes SmoothLLM, a defense that randomizes input perturbations and aggregates predictions to reduce jailbreak attack success rates significantly.
- **Relevance:** Foundational input-perturbation defense technique; widely cited baseline for prompt injection resistance.

### 2. SafeDecoding: Unlocking the Complexity of LLM Jailbreak Defense
- **URL:** https://arxiv.org/abs/2402.06788
- **Summary:** Introduces SafeDecoding, which identifies and mitigates jailbreak attacks by analyzing token-level probabilities during decoding, reducing attack success while preserving generation quality.
- **Relevance:** Decoding-time intervention — a different defense layer from input filtering; useful for multi-layer defense stacks.

### 3. The Instruction Hierarchy: Training LLMs to Prioritize Privileged Instructions
- **URL:** https://arxiv.org/abs/2407.14165
- **Summary:** Proposes the instruction hierarchy framework for training LLMs to recognize and prioritize instructions from trusted sources over untrusted third-party inputs.
- **Relevance:** Core architectural defense — instruction hierarchy is now an industry-standard primitive (OpenAI, Anthropic both implement variants).

### 4. Spotlight: Repositioning Attention for Efficient and Effective LLM Defenses Against Prompt Injection Attacks
- **URL:** https://arxiv.org/abs/2410.10224
- **Summary:** Presents Spotlight, a defense mechanism that repositions attention weights in LLMs to detect and mitigate prompt injection attacks with high detection accuracy and minimal utility impact.
- **Relevance:** Attention-level detection — novel approach that doesn't require input perturbation; low overhead.

### 5. Perimeter Defense: Repurposing In-Context Learning for LLM Security
- **URL:** https://arxiv.org/abs/2406.12539
- **Summary:** Introduces Perimeter Defense, a framework repurposing in-context learning capabilities for security by detecting and filtering adversarial inputs before they reach the core model.
- **Relevance:** Practical defense-in-depth approach using ICL for input screening; compatible with existing LLM deployments.

### 6. LLM Guardrails Are Not Enough: A Comprehensive Study on Bypassing Safety Mechanisms
- **URL:** https://arxiv.org/abs/2501.16743
- **Summary:** Comprehensive study demonstrating that current LLM guardrails can be systematically bypassed, highlighting fundamental vulnerabilities.
- **Relevance:** Critical reality check — demonstrates why single-layer defenses fail; motivates defense-in-depth.

### 7. Adaptive Guardrails: Context-Aware Safety for Large Language Models
- **URL:** https://arxiv.org/abs/2501.19832
- **Summary:** Proposes Adaptive Guardrails, a context-aware safety framework that dynamically adjusts LLM safety interventions based on risk level and context of user interactions.
- **Relevance:** Emerging approach — dynamic guardrails that adapt to conversation context represent the next generation beyond static filters.

### 8. Robust Safety Classifiers: Improving LLM Safety Through Adversarial Training of Input Monitors
- **URL:** https://arxiv.org/abs/2503.07891
- **Summary:** Presents Robust Safety Classifiers, input monitors trained adversarially to detect adversarial prompts and jailbreak attempts, demonstrating improved robustness against adaptive attacks.
- **Relevance:** Adversarial training of classifiers — addresses the arms race where static detectors become obsolete.

### 9. A Survey on Prompt Injection Defense: Taxonomy, Methods, and Open Challenges
- **URL:** https://arxiv.org/abs/2502.12688
- **Summary:** Comprehensive survey of prompt injection defense methods organized into a taxonomy covering input filtering, output monitoring, architectural interventions, and model training.
- **Relevance:** Best starting point for understanding the full landscape of defense approaches; includes open challenges.

### 10. An Empirical Study of Defense Effectiveness Against Prompt Injection Attacks on LLMs
- **URL:** https://arxiv.org/abs/2502.04357
- **Summary:** Large-scale empirical study evaluating existing defense mechanisms against prompt injection attacks, finding many defenses degrade under adaptive attacks and proposing improved evaluation protocols.
- **Relevance:** Essential for realistic defense assessment — demonstrates which defenses hold up under adversarial evaluation and which don't.

## Notable Trends (2025-2026)

1. **Defense-in-depth is the consensus**: Single defenses are fragile; multi-layer architectures combining input detection, instruction hierarchy, and output filtering are now standard recommendation.
2. **Adaptive defenses are emerging**: Static guardrails are being replaced by dynamic systems adjusting based on risk signals and conversation context.
3. **Instruction hierarchy as core primitive**: OpenAI, Anthropic, and others converge on prioritizing system instructions as a fundamental defense.
4. **Gap in production evaluation**: Many defenses work in controlled settings but lack deployment-scale validation data (source #10 specifically documents this).
5. **Adversarial robustness of classifiers**: The arms race between attacks and defensive classifiers is driving adversarial training approaches (source #8).
