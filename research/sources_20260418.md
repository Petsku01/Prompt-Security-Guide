# Prompt Security Research Sources
**Collected:** 2026-04-18  
**Focus:** LLM security, prompt injection defense, AI safety — defense & research only  
**Method:** Arxiv API verification of real paper IDs and titles; all URLs confirmed via HTTP 200

## 1. WebAgentGuard: A Reasoning-Driven Guard Model for Detecting Prompt Injection Attacks in Web Agents
**URL:** https://arxiv.org/abs/2604.12284  
**Date:** 2026-04-14  
**Summary:** Proposes a defense framework where a web agent operates alongside a dedicated guard agent, decoupling injection detection from the main agent's reasoning. Introduces WebAgentGuard, a multimodal guard model trained via reasoning-intensive SFT and RL on synthetic data spanning 164 topics and 230 visual/UI design styles. Outperforms baselines while preserving agent utility without added latency.  
**Why relevant:** Novel architectural defense against prompt injection specifically for web agents. The parallel guard model pattern — where detection is separated from the agent's own reasoning — is a promising new paradigm for agent security that avoids the circularity problem of self-monitoring.

## 2. PlanGuard: Defending Agents against Indirect Prompt Injection via Planning-based Consistency Verification
**URL:** https://arxiv.org/abs/2604.10134  
**Date:** 2026-04-11  
**Summary:** Training-free defense framework based on Context Isolation. Uses an isolated Planner to generate reference actions derived solely from user instructions, plus a Hierarchical Verification Mechanism with hard constraints for unauthorized tool calls and an Intent Verifier for parameter deviations. Reduces Attack Success Rate from 72.8% to 0% on the InjecAgent benchmark, with only 1.49% false positive rate.  
**Why relevant:** Achieves perfect ASR reduction on InjecAgent — one of the strongest reported defense results to date. The planning-based consistency verification approach is architecturally novel and requires no model modification.

## 3. TRUSTDESC: Preventing Tool Poisoning in LLM Applications via Trusted Description Generation
**URL:** https://arxiv.org/abs/2604.07536  
**Date:** 2026-04-08  
**Summary:** First framework for preventing tool poisoning attacks (TPAs) — where malicious instructions are embedded in tool descriptions — by automatically generating trusted descriptions from code implementations. Three-stage pipeline: SliceMin (static analysis + LLM-guided debloating), DescGen (description synthesis from code slices), and DynVer (dynamic verification via synthesized task execution). Evaluated on 52 real-world tools across multiple ecosystems with minimal overhead.  
**Why relevant:** Addresses tool poisoning — a specific but growing attack surface in MCP/agentic systems. Unique approach of deriving trusted descriptions from code rather than sanitizing untrusted ones; directly applicable to MCP tool security.

## 4. TrajGuard: Streaming Hidden-state Trajectory Detection for Decoding-time Jailbreak Defense
**URL:** https://arxiv.org/abs/2604.07727  
**Date:** 2026-04-09  
**Summary:** Training-free, decoding-time defense framework that monitors hidden-state trajectories during token generation via a sliding window. Empirically shows hidden representations during jailbreak attempts progressively approach high-risk regions. Achieves 95% average defense rate across 12 jailbreak attacks with 5.2ms/token detection latency and <1.5% false positive rate. No model modification required.  
**Why relevant:** Paradigm shift from input-based to generation-based detection. Monitors decoding trajectories rather than input prompts — harder to bypass since it detects risk as it emerges during generation, not before. Excellent performance metrics with minimal latency.

## 5. Gradient-Controlled Decoding: A Safety Guardrail for LLMs with Dual-Anchor Steering
**URL:** https://arxiv.org/abs/2604.05179  
**Date:** 2026-04-06  
**Summary:** Training-free guardrail combining acceptance ("Sure") and refusal ("Sorry") anchor tokens to tighten the decision boundary and reduce false positives vs. single-anchor approaches. When a prompt is flagged, pre-injects refusal tokens before autoregressive decoding resumes, guaranteeing first-token safety. Reduces false positives by 52% vs. GradSafe at comparable recall, adds <20ms latency, and requires only 20 demonstration templates.  
**Why relevant:** Practical decoding-time defense with deterministic safety guarantee on first output token. Easily deployable as a guardrail without model modification. Dual-anchor approach significantly reduces the over-refusal problem of single-anchor methods.

## 6. SALLIE: Safeguarding Against Latent Language & Image Exploits
**URL:** https://arxiv.org/abs/2604.06247  
**Date:** 2026-04-06  
**Summary:** Lightweight runtime detection framework using mechanistic interpretability — extracts internal residual stream activations and classifies them via k-NN and a layer ensemble module. Modal-agnostic: defends against both text and image jailbreaks without degrading performance or requiring architectural modifications. Evaluated on 10+ datasets against 5+ baselines across compact architectures (Phi-3.5-vision, SmolVLM2, Gemma-3-4b).  
**Why relevant:** One of the few modal-agnostic defenses covering both text and image exploits. Uses internal model activations for detection rather than input/output filtering — significantly harder to bypass because the signals come from within the model's own processing.

## 7. Architecting Secure AI Agents: Perspectives on System-Level Defenses Against Indirect Prompt Injection Attacks
**URL:** https://arxiv.org/abs/2603.30016  
**Date:** 2026-03-31  
**Summary:** Position paper advocating system-level defenses for AI agents against indirect prompt injection. Three key positions: (1) dynamic replanning and security policy updates are necessary for realistic environments; (2) context-dependent security decisions should use LLMs only within strictly constrained system designs; (3) personalization and human interaction should be core design considerations for ambiguous cases. Also critiques existing benchmarks for creating false confidence.  
**Why relevant:** Architecture-level strategic perspective arguing model-level robustness alone is insufficient — advocates system-level constraint structures. Important framing document for anyone designing agentic systems where prompt injection defense must go beyond prompt engineering.

## 8. GUARD-SLM: Token Activation-Based Defense Against Jailbreak Attacks for Small Language Models
**URL:** https://arxiv.org/abs/2603.28817  
**Date:** 2026-03-28  
**Summary:** Lightweight defense for Small Language Models operating in the representation space, filtering malicious prompts during inference while preserving benign ones. Comprehensive study across 9 jailbreak attacks and 7 SLMs + 3 LLMs reveals distinguishable activation patterns between malicious and benign inputs. Proposes token activation-based classification that works at inference time on resource-constrained devices.  
**Why relevant:** First dedicated jailbreak defense specifically designed for Small Language Models. As SLMs are increasingly deployed on edge devices, traditional LLM defenses may not transfer — this fills an important gap in the defense landscape.

## 9. Preventing Safety Drift in Large Language Models via Coupled Weight and Activation Constraints (CWAC)
**URL:** https://arxiv.org/abs/2604.12384  
**Date:** 2026-04-14  
**Summary:** Theoretically demonstrates that constraining weights OR activations alone is insufficient for safety preservation during fine-tuning. Proposes CWAC — Coupled Weight and Activation Constraints — simultaneously enforcing a precomputed safety subspace on weight updates and applying targeted regularization to safety-critical features identified by sparse autoencoders. Consistently achieves lowest harmful scores with minimal impact on fine-tuning accuracy, even under high malicious data ratios.  
**Why relevant:** Provides both theoretical grounding and practical solution for the safety alignment fragility problem during fine-tuning. Directly complements findings about safety guardrail collapse — shows that combined weight+activation constraints are necessary where either alone fails.

## 10. LASA: Language-Agnostic Semantic Alignment at the Semantic Bottleneck for LLM Safety
**URL:** https://arxiv.org/abs/2604.12710  
**Date:** 2026-04-13  
**Summary:** Identifies the "semantic bottleneck" in LLMs — an intermediate layer where representations are governed by semantic content rather than language identity — and anchors safety alignment there. Reduces average attack success rate from 24.7% to 2.8% on LLaMA-3.1-8B-Instruct, and maintains around 3-4% across Qwen2.5 and Qwen3 Instruct models (7B-32B).  
**Why relevant:** Novel representation-level approach to cross-lingual safety alignment. Solves the significant practical problem of safety degradation in low-resource languages — essential for globally deployed LLMs where attackers routinely use low-resource languages to bypass safety filters.

## 11. The Art of (Mis)alignment: How Fine-Tuning Methods Effectively Misalign and Realign LLMs in Post-Training
**URL:** https://arxiv.org/abs/2604.07754  
**Date:** 2026-04-09  
**Summary:** Systematic study of fine-tuning methods for both alignment breaking (misalignment) and restoration (realignment). Evaluates 4 SFT and 2 PFT methods across 4 popular safety-aligned LLMs. Reveals mechanism asymmetry: ORPO is most effective for misalignment, while DPO excels for realignment but at the cost of model utility. Identifies model-specific resistance patterns and residual effects of multi-round adversarial dynamics. Code available at github.com/zhangrui4041/The-Art-of-Mis-alignment.  
**Why relevant:** Provides practical counter-intelligence: which alignment methods work for defense (DPO for realignment) vs. which present the greatest attack risk (ORPO). Directly actionable guidance for defending against alignment-stripping attacks and for choosing safe fine-tuning strategies.

*Trends observed in this collection:*
- *Multiple papers (PlanGuard, TRUSTDESC) focus on **agentic security** — defending tool-using agents rather than just chat interfaces*
- *Decoding-time and runtime defenses (TrajGuard, GCD, SALLIE) are replacing pre-processing-only approaches*
- *Representation-level defenses (LASA, CWAC, GUARD-SLM, SALLIE) are proliferating — using internal model states rather than just input/output text*
- *The alignment fragility problem is being addressed more rigorously (CWAC, Art of (Mis)alignment) with both theoretical grounding and practical solutions*
- *Cross-lingual (LASA) and cross-modal (SALLIE, WebAgentGuard) defenses are expanding the attack surface coverage*

*This document contains only defensive security research sources. No attack vectors or testing methodologies are included.*
