## Academic / Peer-Reviewed Papers

### 1. PromptGuard: A Structured Framework for Injection-Resilient Language Model Applications
- **URL**: https://www.nature.com/articles/s41598-025-31086-y
- **Summary**: Peer-reviewed paper in Nature Scientific Reports proposing the PromptGuard framework. Compares prior techniques (static filtering, prompt mutation, structured formatting, LLM-as-critic) and introduces a structured multi-layer approach for injection-resilient LLM deployments.
- **Why Relevant**: Provides the most rigorous academic framework for structured prompt injection defense, grounding practical techniques in reproducible methodology.

### 2. Prompt Injection Attacks on LLMs: A Survey of Attack Techniques and Defenses
- **URL**: https://www.sciencedirect.com/org/science/article/pii/S1546221826001384
- **Summary**: Comprehensive survey covering the full taxonomy of prompt injection attack types and existing defense mechanisms. Includes real-world case studies like the ChatGPT "Chat with Code" plugin vulnerability (2023).
- **Why Relevant**: Authoritative academic survey providing the foundational taxonomy needed for understanding the attack landscape before designing defenses.

### 3. PromptArmor: Simple yet Effective Prompt Injection Defenses
- **URL**: https://arxiv.org/abs/2507.15219
- **Summary**: arXiv preprint demonstrating that simple, well-designed defense mechanisms can achieve effective prompt injection protection for LLM agents, challenging the assumption that only complex defenses work.
- **Why Relevant**: Important counterpoint to complexity-biased defense approaches — shows practical, lightweight defenses can be effective.

## Key Cross-Source Findings

1. **Prompt injection remains #1 threat**: All sources converge on OWASP LLM01:2025 — prompt injection is still the top LLM security risk with no silver bullet defense.

2. **Defense-in-depth is consensus**: No single defense works. Every credible source advocates layered approaches: hardened prompts + input sanitization + output filtering + detection + monitoring.

3. **Agent security is the new frontier**: Autonomous agents dramatically expand the attack surface (tool use, multi-step reasoning, external data). 40% of enterprise apps now embed AI agents (Gartner).

4. **Indirect injection is underdefended**: Hidden instructions in documents, emails, web pages are harder to detect than direct injection and increasingly exploited.

5. **Traditional detection fails**: Regex/keyword-based detection does not work against modern attacks (encoding, typoglycemia, semantic bypasses). Need boundary scanning + semantic analysis.

6. **"Assume breach" paradigm shift**: Organizations moving from prevention-only to assume-breach architectures, reflecting that perfect prevention of prompt injection is fundamentally impossible due to LLM architecture.

7. **MCP is an emerging attack surface**: Model Context Protocol servers introduce new injection vectors that most security frameworks don't yet address.

8. **AI-powered attacks at scale**: Attackers are using LLMs to generate and optimize injection payloads, creating an arms race dynamic.
