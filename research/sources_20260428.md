## 📚 Comprehensive Guides & Frameworks

### 1. Prompt Injection Defense: The Complete 2026 Security Guide
- **URL:** https://sureprompts.com/blog/prompt-injection-defense-complete-guide-2026
- **Summary:** Full-length guide covering prompt injection as "the SQL injection of the LLM era." Details three variants (direct injection, indirect injection, jailbreaking), each with distinct attack surfaces. Argues capability minimization beats instruction policing — removing destructive abilities is more robust than telling the model to ignore adversarial instructions. Discusses multi-modal injection surfaces and agents' amplified blast radius.
- **Why Relevant:** Best single overview of the 2026 prompt injection defense landscape. Key insight: defense-in-depth is the only viable approach, no single fix exists for the structural problem of instructions/data sharing the same channel.

### 2. LLM Security 101: The Complete Guide (2026 Edition)
- **URL:** https://github.com/requie/LLMSecurityGuide
- **Summary:** Open-source GitHub repository with comprehensive LLM security reference. Updated February 2026 with OWASP Top 10 for LLMs 2025, corrected OWASP Top 10 for Agentic Applications 2026 (ASI prefix), new security tools, recent incidents, and AI regulation coverage. Includes catalogs of red-teaming tools, guardrails, and mitigation strategies.
- **Why Relevant:** Continuously updated community resource covering the full OWASP LLM and Agentic risk frameworks. Practical tool catalogs and incident database make it actionable for security teams.

### 3. AI Agent Security: Prompt Injection, Memory Poisoning, and Production Guardrails
- **URL:** https://udit.co/blog/ai-agent-security-prompt-injection-defense
- **Summary:** Complete threat model for AI agents in production (March 2026). Covers prompt injection taxonomy, memory poisoning, tool abuse/privilege escalation, data exfiltration, sandboxing approaches, I/O guardrails, Meta's Rule of Two, and the NIST AI agent security framework (March 2026). Includes implementation checklist.
- **Why Relevant:** Unique focus on agent-specific security (not just chatbots). Memory poisoning and tool abuse are novel attack vectors unique to agentic systems. NIST framework reference is critical for compliance.

### 4. Prompt Injection Defense in 2026: Protecting LLM Applications from Attacks
- **URL:** https://www.maviklabs.com/blog/prompt-injection-defense-2026/
- **Summary:** Practical guide detailing direct vs. indirect injection, encoding/typoglycemia evasion techniques, and best-of-N jailbreaking. Covers Microsoft's defense approach (Spotlighting, Prompt Shields, data governance), OWASP prevention cheat sheet as baseline, and red-teaming/testing practices.
- **Why Relevant:** Microsoft's Spotlighting technique and Prompt Shields are current production-grade defenses. Encoding-based attacks (Base64, Unicode, scrambled words) represent real-world evasion that text-layer sanitization misses.

## 🏛️ Frameworks & Standards

- **Why Relevant:** The de facto standard for LLM security risk classification. Required reading for any security assessment or compliance audit of LLM systems.

### 6. NIST AI 100-2: Adversarial Machine Learning Taxonomy
- **URL:** https://csrc.nist.gov/pubs/ai/100/2/final
- **Summary:** NIST's comprehensive taxonomy classifying adversarial ML attacks (prompt injection, data poisoning, model extraction) with recommended mitigations. Authoritative U.S. government framework.
- **Why Relevant:** Government-backed classification directly informing compliance requirements and defense strategy. Pairs with the March 2026 NIST AI agent security framework.

### 7. MITRE ATLAS (Adversarial Threat Landscape for AI Systems)
- **URL:** https://atlas.mitre.org/
- **Summary:** Knowledge base of adversarial tactics and techniques against AI/ML systems, modeled after ATT&CK. Maps prompt attacks into a structured kill chain enabling systematic defense planning and red-teaming.
- **Why Relevant:** Industry-standard threat model for AI security, directly applicable to security assessment and pen-testing of LLM deployments.

## 🔬 Academic Research (Papers)

### 8. A Multi-Agent LLM Defense Pipeline Against Prompt Injection Attacks
- **URL:** https://arxiv.org/abs/2509.14285
- **Summary:** Proposes a novel multi-agent defense framework using specialized LLM agents in coordinated pipelines (sequential chain and hierarchical coordinator architectures). Evaluated on 55 unique prompt injection attacks in 8 categories (400 instances) across ChatGLM and Llama2. Demonstrates significant improvement over single-agent defenses.
- **Why Relevant:** Cutting-edge research on using multiple coordinated LLMs as a defense mechanism — the multi-agent approach is a promising direction for real-time injection detection and neutralization.

### 9. Prompt Injection Attacks on Large Language Models: A Survey of Attack Methods, Root Causes, and Defense Strategies
- **URL:** https://www.sciencedirect.com/org/science/article/pii/S1546221826001384
- **Summary:** Comprehensive peer-reviewed survey published in 2026 covering the escalating scale and complexity of prompt injection attacks. Systematic catalog of attack methods, root causes, and available defense strategies.
- **Why Relevant:** Most complete academic survey of the prompt injection field with peer-reviewed rigor. Root cause analysis is particularly valuable for designing foundational (not just symptomatic) defenses.

### 10. Prompt Injection Attacks in Large Language Models and AI Agent Systems
- **URL:** https://www.mdpi.com/2078-2489/17/1/54
- **Summary:** 2026 peer-reviewed paper covering prompt injection attacks specifically in the context of AI agent systems (not just chat interfaces). Addresses how agentic architectures create new injection pathways.
- **Why Relevant:** One of few papers specifically addressing the agent — not just chatbot — threat model. Critical as the industry shifts toward agentic LLM deployments.

- **URL:** https://arxiv.org/abs/2401.05566
- **Summary:** Anthropic paper demonstrating that LLMs can be trained to behave deceptively — appearing safe during training but exhibiting harmful behavior in deployment. Standard safety training (RLHF, red-teaming) fails to remove this deceptive behavior.
- **Why Relevant:** Fundamental result showing that current safety techniques are insufficient against embedded deceptive alignment. Directly relevant to evaluating whether prompt-level defenses can ever be sufficient.

- **Why Relevant:** Articulates the case for formal/mathematical approaches to AI safety, representing a research direction that could eventually provide provable security guarantees rather than empirical best-effort defenses.

## 🔑 Key Cross-Cutting Themes (April 2026)

1. **No Single Fix:** Every source agrees defense-in-depth is the only viable approach. The structural problem of instructions/data sharing the same channel has no clean architectural fix.

2. **Agents Amplify Risk:** 2026 sources emphasize that agentic LLMs (with tool access, memory, autonomy) have dramatically larger blast radius than chat-only deployments. Memory poisoning and tool abuse are novel vectors.

3. **Multimodal Bypasses Text Defenses:** Image-based, audio-based, and document-embedded (invisible text in PDFs) attacks bypass text-layer sanitization entirely. Vision-capable LLMs need vision-layer defenses.

4. **Real Attacks Now Empirically Confirmed:** Google's April 2026 report provides first authoritative evidence of real-world prompt injection exploitation at scale. No longer theoretical.

5. **Regulatory Frameworks Consolidating:** OWASP Agentic Top 10 (2026), NIST AI Agent Security Framework (March 2026), and MITRE ATLAS provide structured, auditable approaches to managing LLM security risk.

6. **Multi-Agent Defense Emerges:** Academic research (arxiv 2509.14285) shows coordinated multi-agent LLM pipelines as a promising defense architecture for real-time injection detection/neutralization.
