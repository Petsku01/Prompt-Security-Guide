## 🔐 LLM Security Frameworks & Guidelines

### LLM Security 101: The Complete Guide (2026 Edition)
**URL:** https://github.com/requie/LLMSecurityGuide

Includes security tools, case studies, and practical defense strategies. Notable updates include corrected ASI (Agentic Security Issue) prefix identifiers and new security tools like DeepTeam, Promptfoo, and Meta LlamaFirewall.  
**Why Relevant:** Primary reference for current security standards and attack vectors. Contains both offensive and defensive perspectives with practical implementation examples. Covers emerging agentic AI security concerns which are critical for 2026 threat landscape.

### NIST AI Risk Management Framework  
**URL:** https://www.nist.gov/itl/ai-risk-management-framework  
**Brief Summary:** Official NIST framework for managing AI risks, with voluntary guidelines for incorporating trustworthiness into AI systems. Includes companion playbook and specific profile for Generative AI (NIST-AI-600-1) released July 2024.  
**Why Relevant:** Government standard for AI risk management providing structured approach to AI safety. Essential for compliance and enterprise implementations.

## 📈 Emerging Trends Noted

1. **Agentic AI Security:** New category requiring different security approaches than traditional LLMs
2. **RAG-Specific Vulnerabilities:** Vector database poisoning and cross-tenant leakage
3. **Memory Poisoning:** Persistent corruption of agent memory systems
4. **Supply Chain Risks:** Malicious MCP servers and agent plugins
5. **Non-Human Identity (NHI) Abuse:** Exploitation of agent credentials and permissions
