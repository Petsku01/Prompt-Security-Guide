## 1. Academic Papers — Prompt Injection Defenses

| Title | URL | Brief Summary | Why Relevant |
|-------|-----|---------------|--------------|
| "Prompt Injection Attacks and Defenses in LLMs: A Comprehensive Survey" | https://arxiv.org/abs/2310.12835 | Comprehensive survey of prompt injection attack taxonomy and defense strategies | Key reference cataloging all known defense approaches |
| "Prompt Injection Defenses and the Isolation/Utility Tradeoff" (DeepMind) | https://arxiv.org/abs/2402.11196 | Analyzes the fundamental tradeoff between defending against prompt injection and maintaining model utility | Shows no perfect inference-time defense exists; proposes architectural mitigations |
| "SmoothLLM: Defending LLMs Against Jailbreak Attacks" | https://arxiv.org/abs/2310.08106 | Perturbation-based defense against jailbreak prompts | Foundational technique extended by newer work |

| "Spotlight: Detecting Prompt Injection via Token Classification" | https://arxiv.org/abs/2410.01225 | Token-level classification to detect injected prompts in context | Detection-based defense at token granularity |

| "Jailbroken! How Does LLM Safety Training Fail?" (Wei et al.) | https://arxiv.org/abs/2307.02483 | Analyzes why LLM safety training fails and categorizes jailbreak techniques | Widely cited in defense follow-ups |

## Summary of Defense Categories Across Sources

| Category | Description | Key Sources |
|----------|-------------|-------------|
| **Input Validation/Sanitization** | Filtering and encoding user inputs before LLM processing | NIST, OWASP, Rebuff, Lakera |
| **Instruction/Data Separation** | Architecturally separating system instructions from user data | Anthropic, OWASP, DeepMind |
| **Programmable Guardrails** | Runtime constraints on LLM behavior | NVIDIA NeMo, Lakera |
| **Multi-LLM Review Architecture** | Separate LLMs reviewing/auditing outputs | Anthropic, Microsoft |
| **Least-Privilege Tool Access** | Limiting tools/actions an LLM can invoke | OWASP, SANS, Microsoft |
| **Fine-Tuning for Robustness** | Training models to resist injection | Academic papers (SecAlign), DeepMind |
| **Detection/Monitoring** | Real-time detection of injection attempts | Rebuff, Lakera, PyRIT |
| **Red-Teaming** | Proactive testing of defenses | Microsoft PyRIT, NVIDIA Garak, Promptfoo |
| **Formal Verification** | Mathematical guarantees for AI safety | CAIS "Guaranteed Safe AI" |
| **Mechanistic Interpretability** | Understanding model internals for safety | Anthropic, CAIS |
