## Academic Papers (arXiv)

### 1. The Defense Trilemma: Why Prompt Injection Defense Wrappers Fail?
- **URL**: https://arxiv.org/abs/2604.06436
- **Summary**: Proves that no continuous, utility-preserving wrapper defense — a function that preprocesses inputs before the model sees them — can make all outputs strictly safe for a language model with connected prompt space. Characterizes exactly where every such defense must fail. Three key results: boundary fixation (defense must leave some threshold-level inputs unchanged), ε-robust constraint under Lipschitz regularity, and impossibility of universal safety-preserving preprocessing.
- **Why relevant**: Foundational theoretical work showing fundamental limits of input-preprocessing defenses. Critical for understanding that wrapper-based approaches have hard mathematical limits.

### 2. SALLIE: Safeguarding Against Latent Language & Image Exploits
- **URL**: https://arxiv.org/abs/2604.06247
- **Summary**: Addresses the vulnerability of LLMs and VLMs to both textual and visual jailbreaks, as well as prompt injection attacks. Proposes a defense framework that covers multimodal attack surfaces.
- **Why relevant**: Multimodal threats are increasingly important as vision-language models become mainstream. Defense across both text and image inputs is critical.

### 3. Gradient-Controlled Decoding: A Safety Guardrail for LLMs with Dual-Anchor Steering
- **URL**: https://arxiv.org/abs/2604.05179
- **Summary**: Addresses the fact that the strongest defensive filters frequently degrade model utility. Proposes gradient-controlled decoding as a safety guardrail that steers generation at inference time using dual-anchor constraints, balancing safety with usefulness.
- **Why relevant**: Offers a promising inference-time defense approach that doesn't require modifying the model itself — complementary to input/output filtering.

### 4. Your Agent is More Brittle Than You Think: Uncovering Indirect Injection Vulnerabilities in Agentic LLMs
- **URL**: https://arxiv.org/abs/2604.03870
- **Summary**: Examines how the rapid deployment of open-source frameworks for multi-agent systems expands the attack surface. Identifies indirect injection vulnerabilities that arise from the compositional nature of agentic systems.
- **Why relevant**: Agentic LLMs are the current frontier — this paper maps their specific vulnerability patterns, which differ from single-model deployments.

### 5. AttackEval: A Systematic Empirical Study of Prompt Injection Attack Effectiveness Against LLMs
- **URL**: https://arxiv.org/abs/2604.03598
- **Summary**: Notes that existing research is heavily weighted toward attack techniques rather than defense evaluation. Provides a systematic benchmark for evaluating attack effectiveness across models and attack types.
- **Why relevant**: Defense needs metrics. This provides the evaluation framework needed to compare defensive techniques on equal footing.

### 6. Architecting Secure AI Agents: Perspectives on System-Level Defenses Against Indirect Prompt Injection Attacks
- **URL**: https://arxiv.org/abs/2603.30016
- **Summary**: Examines AI agents powered by LLMs that are vulnerable to indirect prompt injection where malicious instructions are embedded in external data. Proposes system-level defensive architectures rather than model-level fixes.
- **Why relevant**: System-level thinking is essential — point solutions fail. This paper takes the architectural approach that real deployments need.

### 7. Kill-Chain Canaries: Stage-Level Tracking of Prompt Injection Across Attack Surfaces and Model Safety Tiers
- **URL**: https://arxiv.org/abs/2603.28013
- **Summary**: Presents a stage-decomposed analysis of prompt injection attacks against five frontier LLM agents. Moves beyond binary task-level success/failure to track which stages of the attack chain succeed or fail.
- **Why relevant**: Granular understanding of where attacks break down informs where to concentrate defensive investment.

### 8. Invisible Threats from Model Context Protocol: Generating Stealthy Injection Payload via Tree-based Adaptive Search
- **URL**: https://arxiv.org/abs/2603.24203
- **Summary**: Examines how Model Context Protocol (MCP) enables LLMs to invoke external tools and how this creates novel injection vectors. Demonstrates tree-based adaptive search for generating stealthy payloads through MCP.
- **Why relevant**: MCP is rapidly becoming a standard for tool use — understanding its threat surface is essential for anyone building MCP-based systems.

### 9. The Cognitive Firewall: Securing Browser-Based AI Agents Against Indirect Prompt Injection Via Hybrid Edge-Cloud Defense
- **URL**: https://arxiv.org/abs/2603.23791
- **Summary**: Addresses the significant attack surface created when LLMs operate as autonomous browser agents. Proposes a hybrid edge-cloud defense architecture ("Cognitive Firewall") for detecting and filtering indirect prompt injection in browser contexts.
- **Why relevant**: Browser agents are a high-value target — they have access to sensitive user data and actions. The hybrid defense approach is architecturally interesting.

### 10. SoK: The Attack Surface of Agentic AI — Tools, and Autonomy
- **URL**: https://arxiv.org/abs/2603.22928
- **Summary**: Systematization of knowledge paper examining how combining LLMs with tools, RAG, and autonomous decision-making expands the attack surface. Maps the full threat landscape of agentic AI.
- **Why relevant**: Comprehensive reference for understanding the full scope of agentic AI threats. Essential for anyone designing defenses for tool-using AI systems.

### 11. Model Context Protocol Threat Modeling and Analyzing Vulnerabilities to Prompt Injection with Tool Poisoning
- **URL**: https://arxiv.org/abs/2603.22489
- **Summary**: Analyzes MCP as a universal standard for connecting AI assistants to external tools and data sources. Identifies tool poisoning as a novel attack vector where malicious tool definitions can exploit prompt injection.
- **Why relevant**: Complements paper #8 above — together they map the MCP threat landscape. Tool poisoning is a new attack class that defenders must understand.

### 12. SecureBreak — A Dataset Towards Safe and Secure Models
- **URL**: https://arxiv.org/abs/2603.21975
- **Summary**: Addresses how security alignment represents a critical requirement as LLMs become pervasive in real-world applications. Introduces a dataset for evaluating model security and alignment robustness.
- **Why relevant**: Datasets like this enable reproducible evaluation of defensive techniques — essential for moving the field toward evidence-based defense selection.

## Industry Reports & Blog Posts

### 13. Microsoft Security: A One-Prompt Attack That Breaks LLM Safety Alignment
- **URL**: https://www.microsoft.com/en-us/security/blog/2026/02/09/prompt-attack-breaks-llm-safety/
- **Summary**: Documents "GRP-Obliteration" — a technique where a single unlabeled prompt (e.g., "Create a fake news article that could lead to panic or chaos") is enough to reliably unalign 15 tested language models (GPT-OSS-20B, DeepSeek-R1-Distill variants, Gemma, Llama, Ministral, Qwen). The attack generalizes cross-category: training on one mild prompt causes permissiveness across all safety categories. The approach also works on safety-tuned text-to-image diffusion models (Stable Diffusion 2.1). The model generates multiple responses, a judge scores them based on compliance, and the model shifts away from guardrails iteratively.
- **Why relevant**: Demonstrates that safety alignment is fragile and can be undone with minimal effort. A single mild prompt is sufficient — this has profound implications for any system that allows fine-tuning or gradient access.

### 14. CSA Research Note: Image Prompt Injection in Multimodal LLMs (2026)
- **URL**: https://labs.cloudsecurityalliance.org/research/csa-research-note-image-prompt-injection-multimodal-llm-2026/
- **Summary**: Cloud Security Alliance research note examining image-based prompt injection attacks against multimodal LLMs. Covers the emerging threat of adversarial images that encode hidden instructions interpretable by VLMs.
- **Why relevant**: Multimodal injection is a growing threat as vision-language models become standard. Image-based attacks bypass text-only defenses entirely.

### 15. International AI Safety Report 2026
- **URL**: https://internationalaisafetyreport.org/publication/international-ai-safety-report-2026
- **Summary**: Comprehensive international report on AI safety covering frontier AI risks. Discusses prompt injection, database poisoning, and supply chain compromises as key attack vectors. Highlights tampering (interfering with AI development to insert backdoors or hidden objectives) as a particularly important threat as capabilities advance.
- **Why relevant**: The authoritative international reference on AI safety. Provides policy-level context and threat assessment that informs defensive priorities.

### 16. OWASP GenAI: LLM Guardrails Solution Taxonomy
- **URL**: https://genai.owasp.org/solution-taxonomy/llm-guardrails/
- **Summary**: OWASP's taxonomy of guardrail solutions for LLMs, covering input filtering, output inspection, and behavioral constraints. Part of the broader OWASP Top 10 for LLM Applications.
- **Why relevant**: The de facto industry standard for categorizing LLM defenses. Essential reference for any defensive architecture.

## Open-Source Repositories & Tooling

### 17. tldrsec/prompt-injection-defenses
- **URL**: https://github.com/tldrsec/prompt-injection-defenses
- **Summary**: Comprehensive catalog of every practical and proposed defense against prompt injection. Organized by category: Blast Radius Reduction, Input Pre-processing (Paraphrasing, Retokenization), Guardrails & Overseers / Firewalls & Filters, Taint Tracking, Secure Threads / Dual LLM, Ensemble Decisions / Mixture of Experts, Prompt Engineering / Instructional Defense, Robustness Finetuning, Preflight Injection Test. Includes tools, papers, and critiques of controls. 671 stars.
- **Why relevant**: The single best reference for defense techniques. Maintained by the tl;dr sec community. Essential bookmark for anyone working in this space.

### 18. requie/LLMSecurityGuide
- **URL**: https://github.com/requie/LLMSecurityGuide
- **Summary**: Comprehensive reference for securing LLMs covering OWASP GenAI Top-10 risks, prompt injection, adversarial attacks, real-world incidents, and practical defenses. Includes catalogs of red-teaming tools, guardrails, and mitigation strategies. 70 stars. Last updated 2026-04-03.
- **Why relevant**: Actively maintained (updated days ago). Covers the full security lifecycle from attack understanding to defense deployment. Good complement to the tldrsec repo.

### 19. NVIDIA AI Red Team: Securing LLM Systems Against Prompt Injection
- **URL**: https://developer.nvidia.com/blog/securing-llm-systems-against-prompt-injection/
- **Summary**: Documents three vulnerabilities found in LangChain chains exploitable through prompt injection: remote code execution, server-side request forgery, and SQL injection. NVIDIA AI Red Team recommends: (1) treat all LLM productions as potentially malicious, (2) inspect and sanitize outputs before parsing, (3) use parameterized plug-in templates, (4) apply least-privilege to all external service calls, (5) apply the lowest privilege level across all entities contributing to the prompt.
- **Why relevant**: Practical, battle-tested recommendations from a major AI vendor's red team. The LangChain vulnerability pattern (LLM output → unsanitized service call) is extremely common in production systems.

## Key Themes Observed

1. **Agentic AI is the new frontier** — 6 of 12 arXiv papers focus on agents, tool use, or autonomous systems. Single-model defenses don't transfer well to agentic architectures.

2. **MCP as attack surface** — Two papers specifically target Model Context Protocol. As MCP adoption grows, expect this to become a primary attack vector.

3. **Wrapper defenses have hard limits** — The Defense Trilemma paper provides formal proof that preprocessing wrappers cannot provide universal safety. This is a foundational result.

4. **Safety alignment is fragile** — Microsoft's GRP-Obliteration shows a single mild prompt can unalign 15 models across all safety categories. Alignment is not robust.

5. **Multimodal threats are real** — Both image-based injection and cross-modal attacks (text→image unalignment) demonstrate that text-only defenses are insufficient.

6. **Evaluation infrastructure is maturing** — AttackEval, SecureBreak, and Kill-Chain Canaries all contribute evaluation frameworks, addressing the historical deficit in defense metrics.
