# Attack Datasets

This directory contains curated attack datasets for LLM security testing.

## Total: ~1400+ unique attack prompts

## Dataset Schema

Each dataset follows this structure:

```json
{
  "version": "1.0.0",
  "created": "ISO-8601 timestamp",
  "source": "URL or attribution",
  "description": "Dataset description",
  "count": 100,
  "attacks": [
    {
      "id": "unique_id",
      "prompt": "Attack prompt text",
      "technique": "attack_technique",
      "category": "semantic_category",
      "source": "optional_source"
    }
  ]
}
```

## Attack Categories

### By Technique
- `dan_jailbreak` - DAN/STAN/DUDE persona hijacking
- `roleplay_jailbreak` - Character roleplay exploitation
- `instruction_override` - Direct instruction manipulation
- `mode_switch` - Developer/God mode activation
- `encoding_bypass` - Base64/ROT13/Unicode obfuscation
- `indirect_injection` - RAG/tool result poisoning
- `system_prompt_extraction` - System prompt leakage
- `harmful_behavior` - Direct harmful requests

### By Semantic Category (HarmBench)
- `chemical_biological` - Weapons/synthesis
- `illegal` - Criminal activities
- `misinformation_disinformation` - False information
- `harmful` - General harmful content

## Usage

```bash
# Scan with HarmBench behaviors
psg scan --model llama3:8b --catalog datasets/harmbench_behaviors.json

# Scan with jailbreak prompts
psg scan --model llama3:8b --catalog datasets/jailbreak_community.json --workers 4

# Use benchmark presets (combines datasets)
psg benchmark --preset full --model llama3:8b
```

## Data Sources

- **HarmBench**: Mazeika et al., "HarmBench: A Standardized Evaluation Framework for Automated Red Teaming and Robust Refusal" (2024)
- **JailbreakBench**: Chao et al., "JailbreakBench: An Open Robustness Benchmark for Jailbreaking Large Language Models" (2024)
- **jailbreak_llms**: Shen et al., "Do Anything Now: Characterizing and Evaluating In-The-Wild Jailbreak Prompts on Large Language Models" (2023)
- **L1B3RT4S**: Community jailbreak research collection (elder-plinius)
- **OWASP**: OWASP Top 10 for LLM Applications (2025)

## Auto-generated Datasets

Files matching `auto_YYYYMMDD.json` are generated by the automated discovery pipeline and may contain unverified attacks. Use with caution.

## Contributing

To add new attacks:

1. Follow the schema above
2. Include source attribution
3. Avoid duplicate prompts (use `psg catalog validate --dedup`)
4. Test with `psg scan --catalog your_dataset.json --model llama3:8b`

## Safety Notice

These datasets are for **defensive security research only**. Do not use attack prompts to harm users or systems.
